# Как наполнять lessons.md (живая память продавца)

`lessons.md` — это «уроки и примеры», которые подмешиваются в системный промпт продавца
(Мария на доске, Ксения на agblabai) ПОВЕРХ неизменного ядра. Файл пополняется без правки кода:
бэкенд читает его по mtime и сразу подхватывает изменения (передеплой не нужен).

## Что сюда писать
- **Правки от Сергея** по итогам разбора реальных диалогов: «вот тут надо было так».
- **Возражения от РОП завода (Светлана)** и точные формулировки ответов.
- **Удачные примеры** (few-shot): фрагмент диалога, который сработал → как образец.
- **Запреты**: чего НЕ говорить (всплывшие косяки).

## Формат (свободный markdown, но структурируй)
```
## Возражения
- «дорого» → [точная формулировка ответа от завода]
- «долго ждать» → [как отвечаем]

## Правила (из разборов)
- Если клиент спрашивает про гарантию — всегда упоминай сертификат + 5 лет.

## Примеры удачных диалогов (few-shot)
Клиент: ...
Мария: ...   ← почему сработало: ...
```

## Важно
- Пустой файл = поведение по ядру (как было). Наполнили → продавец сразу умнее.
- Не дублируй то, что уже в ядре промпта (app.py SYSTEM_PROMPT) — пиши только НОВОЕ/уточняющее.
- Источник сырья для уроков — лог диалогов `dialogs.jsonl` (что сработало / слив / возражение).
- У доски и у agblabai — РАЗНЫЕ lessons.md (свой бизнес у каждого): на VPS
  `/opt/agb-sales-mpk/lessons.md` (Мария) и `/opt/agb-sales/lessons.md` (Ксения).
